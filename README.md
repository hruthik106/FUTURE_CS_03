# API Security Risk Analysis – FUTURE_CS_03

## 📌 Internship Track
Cyber Security (CS)

## 🎯 Objective
To analyze a public API and identify potential security risks, including authentication issues, data exposure, and API misuse.

## 🌐 API Tested
https://jsonplaceholder.typicode.com

## 🛠️ Tools Used
- Postman
- Browser DevTools
- API Documentation

## 🔍 Methodology
- Tested API endpoints using GET requests
- Inspected responses and headers
- Identified security risks
- Classified risk levels
- Provided remediation recommendations

## 🚨 Key Findings
- No Authentication Required
- Excessive Data Exposure
- No Rate Limiting
- Predictable Endpoints

## 📂 Repository Structure
- report.pdf → Final API security report
- screenshots/ → Postman request evidence

## ⚠️ Disclaimer
This analysis was conducted on a public demo API for educational purposes only. No harmful actions were performed.
